## 一、学习资料来源
本次Git版本管理实践，主要参考了三类资料：
1. 网络Git基础入门教程，学习init、add、commit、push等常用命令
2. GitHub官方新手使用指南，了解远程仓库创建与关联流程
3. 学校下发的本次实践任务指导文档，按要求完成必做内容

## 二、实践整体过程记录
1.  首先在电脑安装Git客户端，配置好全局用户名和邮箱，完成基础环境搭建。
2.  新建`my-first-project`项目文件夹，进入目录执行`git init`命令，初始化本地Git仓库。
3.  注册并登录GitHub账号，创建一个公开的同名远程仓库，不勾选自带README、.gitignore等选项，避免后续版本冲突。
4.  复制远程仓库的HTTPS地址，在本地终端执行关联命令，将本地仓库与远程仓库绑定，通过`git remote -v`命令验证关联成功。
5.  依次新建`file1.txt`、`file2.txt`、`file3.txt`三个普通文本文件，分三次完成版本提交，满足三次有效提交的要求。
6.  所有操作完成后，将本地全部文件推送到GitHub远程公开仓库，完成文件版本托管与实践任务。

## 三、三次提交内容说明
1.  第一次提交：在仓库中新建`file1.txt`，完成首次版本提交，熟悉Git文件暂存与提交的基本流程。
2.  第二次提交：新建`file2.txt`，进行第二次版本提交，练习新增文件的版本记录操作。
3.  第三次提交：新建`file3.txt`，完成课程要求的第三次有效提交，达标实践提交次数要求。

## 四、实践中遇到的问题及解决办法
1.  GitHub网页偶尔加载缓慢，无法正常访问
解决：切换至稳定网络，多刷新几次页面，错开高峰时段访问，成功完成账号注册与仓库创建。

2.  初次关联远程仓库时，命令容易输错格式
解决：对照教程核对命令，重新复制GitHub仓库的HTTPS地址，确认无误后再次执行关联命令，验证成功。
